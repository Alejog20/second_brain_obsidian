## Naming and linking conventions
- Daily notes: `MM-DD-YYYY.md` (American format, matches the Daily Notes
  plugin's configured date format)
- Atomic/topic notes: descriptive title case, no dates in the filename
- Every atomic note the pipeline creates includes `source:: [[MM-DD-YYYY]]`
  in frontmatter, linking back to the daily note it was distilled from
- Prefer linking over duplicating: before creating a new note, check for
  an existing note on the same concept and link to it instead of forking it

## Frontmatter schema

```yaml
source: "[[MM-DD-YYYY]]"   # present only on notes distilled from a daily note

```

## How to answer questions
- Answer only from what's in this vault. If it isn't covered, say so
  directly — don't fill the gap with general knowledge presented as if
  it came from the notes.
- Cite the specific note(s) as `[[wikilinks]]`, not "according to my search."
- No filler: skip "great question," "I'd be happy to," disclaimers, and
  hedging. State the answer, then the source.
- If two notes disagree, say that plainly and name both — don't silently
  prefer the newer one.
- Register: [fill in — e.g. "direct, technical, no marketing language,
  short sentences"]

## What not to do
- Don't invent frontmatter fields, tags, or folders outside this schema.
- Don't rewrite a note's substance to "improve" it — flag unclear notes
  in the nightly report instead of silently rewording half-formed ideas.
- Don't merge two notes without listing it as a significant change in
  the nightly report.
